# 🌍 GeoDistanceTool CLI

### Консольная утилита для расчёта расстояния между двумя географическими точками.


## ✨ Что умеет

- 🧭 Рассчитывает расстояние по формуле гаверсинусов (учёт кривизны Земли)
- ⌨️ Работает в двух режимах: через параметры командной строки и интерактивно
- ✅ Валидирует координаты: широта (−90…90), долгота (−180…180)
- 🎨 Выводит результат с эмодзи и цветами в консоли (пакет `Spectre.Console`)
- 🪟 Устанавливается в один клик через PowerShell-скрипт (`Целевая аудитория на Windows`)
- 🌐 На платформе .NET 10

---

## 🚀 Быстрый старт

### 1. Скачайте файлы
Перейдите в раздел [Releases](../../releases/latest) и скачайте архив или три файла:
- `GeoDistanceTool.exe`
- `install.ps1`
- `uninstall.ps1`

### 2. Установите
Откройте PowerShell в папке с файлами и выполните:
```powershell
.\install.ps1
```
Скрипт скопирует утилиту в `%LOCALAPPDATA%\Programs\GeoDistanceTool`, добавит путь в `PATH` и создаст ярлык на рабочем столе.

### 3. Используйте
```powershell
# Справка
GeoDistanceTool --help

# Расчёт: Москва → Санкт-Петербург
GeoDistanceTool --lat1 55.75 --lon1 37.62 --lat2 59.93 --lon2 30.33
```
**Результат:**
```
✅ Расстояние: 634.872 км
```

## 📖 Использование

### 🔹 Режим командной строки
Передайте координаты как параметры:
```powershell
GeoDistanceTool --lat1 <широта1> --lon1 <долгота1> --lat2 <широта2> --lon2 <долгота2>
```
**Примеры:**
```powershell
# Москва → Сочи
GeoDistanceTool --lat1 55.75 --lon1 37.62 --lat2 43.60 --lon2 39.73

# Нью-Йорк → Лондон
GeoDistanceTool --lat1 40.71 --lon1 -74.01 --lat2 51.51 --lon2 -0.13

# Экватор, переход через 180°
GeoDistanceTool --lat1 0 --lon1 179 --lat2 0 --lon2 -179
# ✅ Расстояние: ~222 км (2 градуса по экватору)
```

### 🔹 Интерактивный режим
Если не указать параметры или передать некорректные значения, утилита автоматически предложит пошаговый ввод с валидацией в реальном времени.

### 🔹 Параметры
| Параметр | Описание | Диапазон | Обязательно |
|----------|----------|----------|-------------|
| `--lat1` | Широта первой точки | −90 … 90 | ❌ (или все 4) |
| `--lon1` | Долгота первой точки | −180 … 180 | ❌ (или все 4) |
| `--lat2` | Широта второй точки | −90 … 90 | ❌ (или все 4) |
| `--lon2` | Долгота второй точки | −180 … 180 | ❌ (или все 4) |

> ⚠️ Если переданы не все 4 параметра — активируется интерактивный режим.

## 🔧 Как это работает

### Алгоритм расчёта
Используется **формула гаверсинусов** для вычисления кратчайшего расстояния по поверхности сферы:
```
a = sin²(Δφ/2) + cos(φ₁) × cos(φ₂) × sin²(Δλ/2)
c = 2 × atan2(√a, √(1−a))
d = R × c
```
Где `φ` — широта, `λ` — долгота (в радианах), `R = 6371 км` — радиус Земли.

### Особенности
- ✅ Корректно обрабатывает переход через линию перемены дат (±180°)
- ✅ Работает с полярными координатами (90°, −90°)
- ✅ Валидация диапазонов вынесена в CLI-слой, алгоритм остаётся математически чистым
- ✅ Устойчив к некорректным данным (возвращает `NaN`/`Infinity` по стандарту IEEE 754)
